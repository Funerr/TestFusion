---
name: recover-device-session
description: Use when 需要在应用切后台、崩溃重拉或异常弹窗后恢复测试现场，并继续执行后续步骤。
call_mcp: true
parameters:
  package:
    type: string
    required: false
    description: 目标应用包名，未传时沿用当前运行配置
  activity:
    type: string
    required: false
    description: 目标应用首页 Activity，未传时沿用当前运行配置
  verify:
    type: boolean
    required: false
    description: 是否在恢复后校验前台页面，默认 true
---

# Implementation

1. 先按 Home 让应用切到后台，必要时清理遮挡弹窗。
2. 执行 `mobile_launch_app` 重新拉起目标应用。
3. 观察前台页面、关键控件或断言结果，确认设备状态已恢复到可继续执行的状态。
