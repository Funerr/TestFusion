---
name: handle-permission-dialog
description: Use when 需要在鸿蒙 4.2 设备上处理系统权限弹窗，并根据允许、仅本次允许或拒绝策略执行点击。
call_mcp: true
parameters:
  decision:
    type: string
    required: true
    description: 处理策略，可选 allow、allow_once、allow_while_using 或 deny
  permission_hint:
    type: string
    required: false
    description: 权限类型提示，如 位置、相机、麦克风
  verify:
    type: boolean
    required: false
    description: 是否校验弹窗消失及权限状态，默认 true
---

# Implementation

1. 通过观察能力识别当前权限弹窗和按钮文案。
2. 执行 `mobile_handle_permission_dialog`，按 `decision` 选择对应按钮。
3. 通过 `mobile_assert_permission_dialog` 或页面状态确认弹窗已经消失。

# Examples

## Example 1 - 允许位置权限一次
**Input:**
```json
{"decision":"allow_once","permission_hint":"位置","verify":true}
```
