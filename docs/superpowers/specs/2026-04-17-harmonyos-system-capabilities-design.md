# 鸿蒙 4.2 高频系统能力设计

## 目标

为华为手机 / 鸿蒙 4.2 的自动化测试场景补充高频系统 skill、Cursor rules 和 MCP 工具，使系统操作可以被：

1. 中文测试步骤识别
2. 编译成 DSL 动作
3. 展开成 MCP 步骤
4. 执行并校验系统状态

## 范围

### 系统开关

- WLAN
- 蓝牙
- 定位
- 移动数据
- 飞行模式
- NFC
- 个人热点
- 深色模式
- 自动旋转

### 系统数值

- 亮度
- 媒体音量
- 铃声音量

### 系统交互

- 控制中心
- 通知中心
- 系统设置首页
- 系统设置二级页
- 权限弹窗处理

## 设计原则

- Skill 和 rules 尽量中文
- MCP 仍保持统一 `mobile_*` 命名
- 系统状态与设备状态共用一套状态模型
- simulation backend 与 adb backend 都可工作
- 断言优先基于系统状态，不只依赖截图
